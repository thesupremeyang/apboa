# 模型供应商管理

模型供应商是智能体的"推理引擎"，负责管理不同 AI 服务提供商的连接信息和认证配置。平台通过统一的供应商抽象层，支持接入多种主流大模型服务，并对每个供应商下的模型提供完整的生命周期管理与连接性检测能力。


## 功能概述

- 支持 4 种供应商类型：DashScope、OpenAI、Anthropic、Ollama
- 每个供应商下可配置多个模型，以**卡片网格**形式展示
- 支持两种认证方式：直接配置 `API Key`、`环境变量`引用
- 模型具备**六态连接检测**系统：停用 / 待检测 / 检测中 / 已连接 / 检测失败
- 支持模型类型：文本、图像、音频、视频
- 模型支持流式输出和深度思考模式
- 供应商下的模型可独立管理，支持启用/禁用、测试连接


## 核心概念

### 供应商类型

| 类型 | 说明 | 默认 Base URL |
|------|------|---------------|
| **DashScope** | 阿里云百炼平台 | `https://dashscope.aliyuncs.com` |
| **OpenAI** | OpenAI 官方 API | `https://api.openai.com` |
| **Anthropic** | Anthropic Claude API | `https://api.anthropic.com` |
| **Ollama** | 本地 Ollama 服务 | `http://localhost:11434` |

### 认证方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **直接配置（CONFIG）** | 在平台上直接填写 API Key，系统加密存储 | 快速配置，开发调试 |
| **环境变量（ENV）** | 填写环境变量名，运行时从部署环境读取 | 更安全，适合生产环境 |

### 模型配置属性

每个供应商下可配置多个模型，每个模型包含以下属性：

| 属性 | 说明 | 示例 |
|------|------|------|
| 模型名称 | 卡片展示名称 | `GPT-4o` |
| 模型编号 | API 调用标识符 | `gpt-4o`、`qwen-max` |
| 模型类型 | 文本/图像/音频/视频，支持多选 | `CHAT`、`IMAGE` |
| 是否支持流式 | 是否支持流式输出响应 | 是/否 |
| 是否支持思考 | 是否支持深度推理思考模式 | 是/否 |
| 上下文窗口 | 最大上下文 Token 数 | `128000` |
| 最大输出 | 单次对话最大输出 Token 数 | `4096` |
| 温度 | 生成随机性参数（0~2），越高越随机 | `0.7` |
| Top-P | 核采样参数（0~1） | `0.9` |
| Top-K | Top-K 采样参数 | `50` |
| 重复惩罚 | 重复内容惩罚系数 | `1.0` |
| 随机种子 | 可复现生成的种子值 | `42` |


## 模型卡片界面

重构后的模型管理采用**全屏卡片式布局**，替代了原有的表格列表形式。

### 卡片结构

![模型卡片占位](images/model-cards.png)

每张卡片包含以下区域：

| 区域 | 内容 | 说明 |
|------|------|------|
| **头像区** | 模型图标 + 角标 | 根据连接状态变换头像色彩和角标 |
| **标题行** | 模型名称 | 可点击编辑，悬浮展示模型编号；超长自动省略 |
| **操作菜单** | 三点下拉菜单 | 测试连接 / 编辑 / 启用禁用 / 删除 |
| **描述区** | 模型描述 | 最多两行，超出省略 |
| **参数行** | Context / Tokens / T | 展示上下文窗口、最大输出、温度参数 |
| **标签区** | 流式/思考 + 模型类型 | 左侧显示流式/思考状态，右侧显示模型类型 |

### 连接状态六态系统

每张卡片左上角的头像及右下角角标实时反映模型连接状态，共六种状态：

| 状态 | 头像背景色 | 图标色 | 角标 | 说明 |
|------|-----------|--------|------|------|
| **已停用** | 灰 `#e7e7e7` | 灰 `#757575` | 无 | 模型已被禁用 |
| **待检测** | 黄 `#fff8e1` | 橙 `#ffa726` | 黄色叹号 ⚠ | 尚未进行连接测试 |
| **检测中** | 蓝 `#e3f2fd` | 蓝 `#42a5f5` | 无（loading 旋转） | 连接测试正在进行 |
| **已连接** | 绿 `#e8f5e9` | 绿 `#52c41a` | 绿色对号 ✓ | 连接测试通过 |
| **检测失败** | 红 `#ffebee` | 红 `#ef5350` | 红色叉号 ✗ | 连接测试失败，悬浮查看失败原因 |

> **提示**：鼠标悬浮在头像上可以查看当前状态的详细说明；检测失败时会显示服务端返回的具体错误信息。

### 工具栏

页面顶部工具栏采用**吸顶设计**（滚动时固定在顶部），包含：

- **新增模型** — 打开模型配置表单
- **搜索框** — 按模型名称搜索，支持回车触发

搜索为精确匹配，结果替换当前展示的模型列表。


## 操作指南

### 创建供应商

1. 在模型供应商管理页面点击 **添加模型供应商** 卡片
2. 填写表单：
   - **类型**：选择供应商类型（选择后自动填充默认 Base URL）
   - **名称**：供应商名称
   - **描述**：供应商说明
   - **Base URL**：API 地址
   - **认证方式**：直接配置或环境变量
   - **API Key / 环境变量名**：根据认证方式填写
3. 点击确认提交

### 打开模型管理

在供应商卡片右上角菜单点击 **配置模型**，弹出全屏模型管理窗口，展示该供应商下所有模型卡片。

### 新增模型

1. 点击顶部工具栏的 **新增模型** 按钮
2. 填写模型配置表单（名称、编号、类型、参数等）
3. 点击确认提交后，卡片列表自动刷新

### 编辑模型

1. 点击模型卡片标题，或在右侧下拉菜单中选择 **编辑**
2. 修改模型配置
3. 点击确认提交

### 测试连接

1. 在模型卡片右上角菜单中点击 **测试连接**
2. 头像区立即变为蓝色 loading 态
3. 测试完成后卡片列表自动刷新，头像和角标更新为最新状态
4. 测试失败时，鼠标悬浮头像可查看具体错误信息

### 启用/禁用模型

在卡片右上角菜单中选择 **启用** 或 **禁用**：

- 禁用的模型头像变为灰色，角标消失
- 当前被智能体引用的模型禁用时，弹出二次确认并提示影响的智能体列表

### 删除模型

在卡片右上角菜单中选择 **删除**：

- 被智能体引用的模型删除时，弹出二次确认并列出引用智能体
- 删除后卡片列表自动刷新

:::warning
删除或修改被智能体使用的模型时，关联智能体将自动重新注册。
:::

### 查看供应商详情

点击供应商卡片右上角菜单 → **查看**，可查看供应商完整信息。

### 删除供应商

点击供应商卡片右上角菜单 → **删除**。

:::warning
删除供应商前需确认其下没有被智能体使用的模型。系统会进行二次确认。
:::

### 搜索模型

在顶部工具栏搜索框中输入模型名称，回车后按名称精确匹配筛选。清空搜索框后列表恢复全部展示。


## 在智能体中使用

在智能体配置的「模型与提示词」步骤中：

1. 选择供应商（按类型分类展示）
2. 选择该供应商下的模型
3. 可选配置模型参数覆盖（温度、Top-P 等）

## 常见问题

### 如何接入 Ollama 本地模型？

1. 创建类型为 `Ollama` 的供应商
2. Base URL 填写 Ollama 服务地址（默认 `http://localhost:11434`）
3. 认证方式选择"直接配置"，API Key 可留空或填写任意值
4. 在模型管理中添加本地已下载的模型编号
5. 通过「测试连接」验证连通性

### 如何接入兼容 OpenAI 协议的第三方服务？

1. 创建类型为 `OpenAI` 的供应商
2. 修改 Base URL 为第三方服务地址
3. 填写对应的 API Key
4. 通过「测试连接」验证连通性

### 环境变量认证如何使用？

1. 认证方式选择"环境变量"
2. 填写环境变量名（如 `OPENAI_API_KEY`）
3. 在部署环境中设置对应的环境变量值
4. 系统运行时自动从环境变量读取 API Key

### 连接测试失败怎么办？

1. 鼠标悬浮在模型卡片的头像上查看具体错误信息
2. 常见原因：API Key 无效、Base URL 不可达、模型编号不存在
3. 请检查供应商的 Base URL 和认证配置是否正确
4. 修正后重新执行「测试连接」
