# MCP 服务管理

MCP（Model Context Protocol）管理是智能体的"外部能力接入中枢"，通过标准化协议将异构的外部工具、服务、数据源统一封装为智能体可理解和调用的能力，实现"思考-行动"闭环。


## 功能概述

- 支持三种协议类型：HTTP、SSE、STDIO
- 支持同步和异步两种运行模式
- 自动健康检查，监控 MCP 服务器状态
- 智能体可关联多个 MCP 服务器


## 核心概念

### 协议类型

| 协议 | 说明 | 配置项 |
|------|------|--------|
| **HTTP** | 标准 HTTP 请求调用 MCP 服务 | URL、请求头、查询参数 |
| **SSE** | 基于 Server-Sent Events 的流式连接 | URL、请求头、查询参数 |
| **STDIO** | 通过标准输入输出与本地进程通信 | 命令、参数、环境变量、工作目录、编码 |

### 运行模式

| 模式 | 说明 |
|------|------|
| **同步（SYNC）** | 工具调用同步等待结果 |
| **异步（ASYNC）** | 工具调用异步返回结果 |

### 健康状态

| 状态 | 说明 |
|------|------|
| **HEALTHY** | 服务器正常运行 |
| **UNHEALTHY** | 服务器异常 |
| **UNKNOWN** | 未进行健康检查 |


## 操作指南

### 创建 MCP 服务器

1. 在 MCP 管理页面点击 **添加新 MCP 服务器** 卡片
2. 填写表单：
   - **名称**：服务器名称
   - **描述**：服务器功能说明
   - **协议类型**：HTTP / SSE / STDIO
   - **运行模式**：同步 / 异步
   - **超时时间**：调用超时时间（毫秒）
   - **协议配置**：根据协议类型填写

3. HTTP/SSE 协议配置：
   - **URL**：服务地址
   - **请求头**：自定义请求头键值对
   - **查询参数**：URL 查询参数键值对

4. STDIO 协议配置：
   - **命令**：可执行文件路径
   - **参数**：命令行参数列表
   - **环境变量**：运行时环境变量键值对
   - **工作目录**：进程工作目录
   - **编码**：字符编码（默认 UTF-8）

### 查看详情

点击卡片右上角菜单 → **查看**，查看完整配置和健康状态。

### 编辑配置

点击卡片右上角菜单 → **编辑**，修改服务器配置。

:::warning
修改 MCP 服务器配置后，关联智能体将自动重新注册。
:::

### 删除服务器

点击卡片右上角菜单 → **删除**。

### 搜索与筛选

- **协议筛选**：页面顶部分段控制器切换协议类型
- **名称搜索**：输入关键词后回车搜索


## 在智能体中使用

在智能体配置的「知识库与 MCP」步骤中：

1. 在 **MCP 服务器** 区域，按协议类型折叠面板浏览
2. 勾选需要关联的 MCP 服务器（Checkbox 多选）

## 常见问题

### HTTP 和 SSE 协议如何选择？

- **HTTP**：适用于简单的请求-响应模式，服务端无需保持连接
- **SSE**：适用于需要服务端推送或长连接的场景

### STDIO 模式的安全注意事项

- STDIO 模式会在服务器本地启动进程
- 确保命令路径安全，避免执行危险命令
- 环境变量中不要包含敏感信息（使用平台的环境变量管理）

### 超时时间如何设置？

- 建议根据工具的响应时间合理设置
- 简单查询类工具：5~10 秒
- 复杂处理类工具：30~60 秒
- 长时间任务：可使用异步模式
