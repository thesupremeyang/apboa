# 智能体管理

智能体是 AI 系统的核心组件，通过配置模型、工具、提示词、敏感词、技能包、MCP 和知识库等多个维度，赋予智能体强大的感知、推理和执行能力。精心设计的智能体能够理解复杂需求、制定执行计划、调用各类工具和知识，最终高效地完成用户目标。

## 功能概述

- 支持两种智能体类型：自定义（CUSTOM）和 A2A（Agent-to-Agent）
- 自定义智能体支持多步骤向导式创建，涵盖模型、工具、知识库等全方位配置
- A2A 智能体支持 WellKnown 和 Nacos 两种发现模式
- 提供配置面板，包含编辑、定时任务、架构图、API 文档、日志、统计分析
- 支持启用/禁用、搜索筛选、标签分类
- 支持架构可视化图，直观展示智能体关联的资源
- 支持记忆、计划、结构化输出等高级功能

## 核心概念

### 智能体类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| **自定义（CUSTOM）** | 本地 ReAct 智能体，可完整配置模型、工具、知识库等 | 需要精细控制智能体行为的场景 |
| **A2A** | 远程智能体，通过 A2A 协议连接外部 Agent | 已有外部 Agent 服务，需通过 A2A 协议集成 |

### 工具选择策略

| 策略 | 说明 |
|------|------|
| **AUTO** | 自动选择 — 模型自行判断是否调用工具 |
| **NONE** | 禁止调用 — 不允许调用任何工具 |
| **REQUIRED** | 强制调用 — 必须调用至少一个工具 |
| **SPECIFIC** | 强制调用指定工具 — 必须调用指定的某个工具 |

### 结构化输出提醒模式

| 模式 | 说明 |
|------|------|
| **TOOL_CHOICE** | 通过工具选择机制实现结构化输出 |
| **PROMPT** | 通过提示词引导实现结构化输出 |

### A2A 发现模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **WELLKNOWN** | 通过 Well-Known 路径发现 Agent Card | 已知 Agent 的 Base URL 和 Card 路径 |
| **NACOS** | 通过 Nacos 注册中心发现 Agent | Agent 注册到 Nacos 服务 |

## 操作指南

### 创建自定义智能体

在智能体管理页面点击 **添加新智能体** 卡片 → 选择 **自定义** 类型，进入多步骤向导：

#### 步骤一：基本信息

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| **名称** | 智能体名称 | 是 |
| **智能体编号** | 智能体的唯一代码标识 | 是 |
| **描述** | 智能体功能说明 | 否 |
| **标签** | 分类标签，用于分组管理 | 否 |

![基本信息占位](images/agent-form-basic.png)

#### 步骤二：模型与提示词

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| **模型配置** | 选择基础模型配置（从模型供应商管理中已配置的模型列表选择） | 是 |
| **模型参数覆盖** | 覆盖基础模型参数（如温度、Top-P 等），不填则使用模型默认值 | 否 |
| **提示词模板** | 选择系统提示词模板（从提示词模板管理中选择） | 否 |
| **跟随模板** | 是否跟随模板内容变化（开启后模板更新时智能体的系统提示词同步更新） | 否 |
| **系统提示词** | 自定义系统提示词内容（未选模板时手动编写，选了模板则自动填充） | 否 |

![模型与提示词占位](images/agent-form-model.png)

:::tip
选择提示词模板并开启「跟随模板」后，系统提示词将自动同步模板内容，无需手动维护。若需自定义，可关闭跟随模板后手动编写。
:::

#### 步骤三：工具与能力

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| **钩子** | 关联钩子（Hook），在智能体执行前后触发自定义逻辑 | 否 |
| **工具选择策略** | AUTO / NONE / REQUIRED / SPECIFIC | 否（默认 AUTO） |
| **工具** | 关联工具（Tool），为智能体提供执行能力 | 否 |
| **指定工具名称** | 当策略为 SPECIFIC 时，指定必须调用的工具名称 | 条件必填 |
| **技能包** | 关联技能包（Skill），为智能体提供预置能力组合 | 否 |
| **执行环境配置** | 关联执行环境配置，允许智能体执行代码 | 否 |
| **敏感词配置** | 关联敏感词配置 | 否 |
| **启用敏感词过滤** | 是否启用敏感词检测与过滤 | 否 |

![工具与能力占位](images/agent-form-tools.png)

#### 步骤四：知识库与 MCP

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| **知识库** | 关联知识库，为智能体提供专业知识检索能力 | 否 |
| **MCP 服务器** | 关联 MCP 服务器，扩展智能体的工具调用能力 | 否 |
| **子智能体** | 关联子智能体，实现多 Agent 协作 | 否 |

![知识库与MCP占位](images/agent-form-knowledge.png)

:::warning
子智能体不能形成循环引用，即智能体 A 不能是智能体 B 的子智能体的同时，智能体 B 又是智能体 A 的子智能体。
:::

#### 步骤五：高级设置

| 字段 | 说明 | 默认值 |
|------|------|--------|
| **启用计划** | 是否启用 ReAct 计划模式，让智能体先制定执行计划再逐步执行 | 否 |
| **最大迭代次数** | ReAct 循环的最大迭代次数 | 50 |
| **最大子任务数** | 计划模式下的最大子任务数 | 10 |
| **计划需要确认** | 生成的计划是否需要用户确认后才执行 | 否 |
| **显示工具调用过程** | 是否在对话中展示工具调用详情 | 是 |
| **启用记忆** | 是否启用对话记忆功能 | 是 |
| **启用记忆压缩** | 是否启用记忆压缩（长对话时自动压缩历史） | 否 |
| **记忆压缩配置** | 记忆压缩的参数配置 | - |
| **执行环境配置** | 该配置可开启 [工作空间]，同时可赋予智能体执行 Shell 脚本、读取&写入文件的能力 | - |
| **Studio 配置** | 关联 Studio 画布配置 | - |

![高级设置占位](images/agent-form-advanced.png)

:::tip
- 启用计划模式适合复杂任务场景，智能体会先分析任务并制定执行计划
- 启用记忆可以让智能体在多轮对话中保持上下文连贯
- 结构化输出适合需要智能体返回特定格式数据的场景（如 JSON 表单）
:::

### 创建 A2A 智能体

在智能体管理页面点击 **添加新智能体** 卡片 → 选择 **A2A** 类型：

#### 基本信息

与自定义智能体相同，填写名称、编号、描述、标签。

#### A2A 配置

**WellKnown 模式：**

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| **Agent 名称** | 远程 Agent 的名称标识 | 是 |
| **Base URL** | 远程 Agent 服务的 Base URL | 是 |
| **Agent Card 路径** | Agent Card 文件的相对路径（如 `/.well-known/agent.json`） | 是 |
| **认证头** | HTTP 请求的认证头信息（Key-Value 对，支持环境变量引用） | 否 |

![A2A WellKnown配置占位](images/agent-a2a-wellknown.png)

**Nacos 模式：**

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| **Agent 名称** | 注册到 Nacos 的 Agent 名称 | 是 |
| **Nacos 属性** | Nacos 连接属性（Key-Value 对，支持环境变量引用） | 是 |

![A2A Nacos配置占位](images/agent-a2a-nacos.png)

:::tip
认证头和 Nacos 属性中的值支持环境变量引用：开启「环境变量」开关后，值字段填写环境变量的 Key，运行时将自动从系统环境变量中获取实际值。这种方式避免了在配置中明文存储密钥等敏感信息。
:::

### 查看详情

点击卡片右上角菜单 → **查看**，查看智能体完整配置：

- 自定义智能体：展示基本信息、模型配置、关联资源数量等
- A2A 智能体：展示基本信息、A2A 连接配置、认证头信息等

### 设计面板

点击卡片上的 **设计** 按钮，进入智能体配置面板，面板包含以下功能页签：

| 页签 | 说明 | 可用类型 |
|------|------|----------|
| **配置** | 编辑智能体的完整配置（自定义 / A2A 各有专属编辑界面） | 自定义 + A2A |
| **定时** | 配置定时任务，让智能体定时自动执行 | 自定义 + A2A |
| **架构** | 查看智能体架构可视化图，直观展示关联的所有资源 | 仅自定义 |
| **API** | 查看智能体的 API 文档和调用方式 | 自定义 + A2A |
| **日志** | 查看智能体的对话历史日志 | 自定义 + A2A |
| **统计** | 查看智能体的使用统计和性能指标 | 自定义 + A2A |

### 架构图

架构图以可视化方式展示自定义智能体关联的所有资源，中心节点为当前智能体，周围展示：

- 模型节点
- 提示词节点
- 钩子节点
- 工具节点
- 技能包节点
- MCP 节点
- 知识库节点
- 子智能体节点
- 高级配置节点

![架构图占位](images/agent-architecture.png)

### 启用/禁用智能体

- 通过卡片上的开关切换启用状态
- 禁用后智能体将从运行时注销，不再响应请求
- 启用后智能体将重新注册到运行时

:::warning
如果智能体关联了定时任务，需先禁用定时任务才能禁用智能体。
:::

### 删除智能体

点击卡片右上角菜单 → **删除**：

:::warning
删除前需确保：
1. 智能体没有关联的定时任务（需先解绑定时任务）
2. 删除后不可恢复，请谨慎操作
:::

### 搜索与筛选

- **类型筛选**：页面顶部分段控制器切换「全部 / 自定义 / A2A」
- **标签筛选**：选择标签过滤智能体
- **名称搜索**：输入关键词后回车搜索

## 配置说明

### 模型参数覆盖

模型参数覆盖允许在基础模型配置上调整参数，支持以下字段：

| 参数 | 说明 |
|------|------|
| temperature | 生成温度，控制随机性（0.0 ~ 2.0） |
| topP | 核采样概率阈值 |
| topK | Top-K 采样数量 |
| maxTokens | 最大生成 Token 数 |
| repeatPenalty | 重复惩罚系数 |
| seed | 随机种子 |

### 认证头配置（KvMap）

A2A 智能体的认证头和 Nacos 属性使用 KvMap 结构：

| 字段 | 说明 |
|------|------|
| **key** | 属性名 / Header 名 |
| **value** | 属性值 / Header 值（或环境变量 Key） |
| **evn** | 是否为环境变量引用 |

示例：
- 直接值：`key=Authorization`, `value=Bearer xxx`, `evn=false`
- 环境变量：`key=Authorization`, `value=API_AUTH_TOKEN`, `evn=true`（运行时从环境变量 `API_AUTH_TOKEN` 读取）

### 版本号

每个智能体都有一个版本号（如 `1.0.0`），用于跟踪配置变更。

## API 接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/agent/definition/page` | 分页查询智能体 | - |
| GET | `/agent/definition/{id}` | 获取智能体详情 | SkAccess / ChatKeyAccess |
| POST | `/agent/definition` | 新增智能体 | ADMIN / EDIT |
| PUT | `/agent/definition` | 修改智能体 | ADMIN / EDIT |
| DELETE | `/agent/definition` | 删除智能体 | ADMIN / EDIT |
| POST | `/agent/definition/used-with-agent` | 查询被哪些 Agent 使用 | - |
| GET | `/agent/definition/get/tags` | 获取所有标签 | - |
| GET | `/agent/definition/{id}/allow/file-type` | 获取允许的文件类型 | SkAccess / ChatKeyAccess |
| POST | `/agentA2a` | 保存 A2A 配置 | - |
| GET | `/agentA2a/{agentId}` | 获取 A2A 配置 | - |

### 查询参数

| 参数 | 说明 | 条件 |
|------|------|------|
| name | 智能体名称（模糊匹配） | 可选 |
| agentType | 智能体类型（CUSTOM / A2A） | 可选 |
| agentCode | 智能体编号（精确匹配） | 可选 |
| tag | 标签（精确匹配） | 可选 |
| enabled | 是否可用 | 可选 |

## 常见问题

### 智能体无法启用？

- 检查是否已配置模型（模型配置为必填项）
- 检查关联的定时任务是否已禁用（需先禁用定时任务才能修改智能体）

### A2A 智能体连接失败？

- 检查 Base URL 是否正确可达
- 检查 Agent Card 路径是否正确
- 检查认证头配置是否正确
- 如果使用环境变量，确认系统环境变量已正确设置

### 工具未被调用？

- 检查工具选择策略是否为 NONE（禁止调用）
- 如果策略为 SPECIFIC，检查指定工具名称是否正确
- 检查工具是否已启用

### 智能体响应不符合预期？

- 调整系统提示词，使指令更加清晰明确
- 检查是否启用敏感词过滤，可能误过滤了内容
- 调整模型参数（如增大 temperature 增加创造性，降低 temperature 增加确定性）
- 检查关联的知识库检索是否返回了正确结果

### 删除智能体提示"请先解绑定时任务"？

- 进入设计面板 → 定时页签
- 禁用或删除关联的定时任务
- 然后重新执行删除操作
