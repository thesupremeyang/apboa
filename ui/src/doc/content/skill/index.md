# 技能包管理

技能包是将相关工具、参考文档、示例和脚本打包为可复用技能单元的管理机制，赋予智能体完成特定领域复杂任务的能力。技能包以**文件目录**形式组织，支持在平台内置编辑器中直接管理文件结构。


## 功能概述

- 基于文件目录的技能包结构，支持在线编辑与管理
- 左侧文件树 + 右侧编辑器的双栏交互模式
- 支持新建文件/文件夹、上传文件、右键菜单操作
- 文件类型白名单管控，非文本文件引导下载查看
- 支持三种导入模式：本地装载、ZIP 压缩包、Git 仓库
- 技能包可关联多个工具，运行时自动注册到工具集
- SKILL.md 元信息与数据库自动双向同步


## 技能包目录结构

每个技能包在文件系统中为一个独立目录，标准结构如下：

```
<技能包名称>/
  SKILL.md            # 技能定义文件（必需），含 YAML frontmatter
  scripts/            # 可执行脚本目录
  examples/           # 示例文件目录
  references/         # 参考文档目录
```

### SKILL.md 文件

`SKILL.md` 是技能包的元信息文件，须包含 YAML frontmatter：

```yaml
---
name: 技能包名称
description: 技能包描述
---
# 正文内容（Markdown）
```

- `name` 字段作为技能包唯一标识，修改后自动同步到数据库
- `description` 字段描述技能功能，修改后自动同步到数据库
- 正文部分使用 Markdown 编写，用于描述技能能力、使用场景和操作步骤
- SKILL.md 不允许删除，不支持右键菜单操作

### 附属目录

| 目录 | 用途 | 说明 |
|------|------|------|
| `scripts/` | 可执行脚本 | 存放 Python、Shell、JavaScript 等脚本文件 |
| `examples/` | 示例文件 | 存放技能使用示例和演示文件 |
| `references/` | 参考文档 | 存放参考文档、知识片段等 |

以上三个目录及其下的文本文件会自动入库管理；也支持在根目录或任意子目录下创建自定义文件。


## 创建与管理技能包

### 手动创建

1. 在技能包管理页面点击 **添加新技能包** 卡片
2. 填写基本信息：名称、描述、分类
3. 点击创建后自动进入技能包编辑器，系统已初始化 `SKILL.md` 文件

### 技能包编辑器

编辑器采用**左侧文件树 + 右侧编辑器**的双栏布局：

**工具栏**：
- **返回** — 返回技能包列表
- **新建文件**（+）— 在根目录或选定目录下创建文件
- **新建文件夹**（文件夹图标）— 创建目录，提供 `scripts`、`references`、`examples` 快捷预设
- **下载技能包** — 将整个技能包目录打包为 ZIP 下载

**文件树操作**（右键菜单）：
- **新建文件** / **新建文件夹** — 在指定目录下创建
- **上传文件** — 上传本地文件到当前目录，文本文件自动入库，二进制文件直接落盘
- **删除** — 删除文件或文件夹（目录删除时会自动清理数据库关联记录）

**文件管理特性**：
- 文件树默认全部展开，方便浏览完整结构
- 删除当前正在编辑的文件后，编辑区自动清空
- 文件类型在白名单内的可直接在线编辑（如 `.md`、`.py`、`.js`、`.json` 等）
- 非白名单文件类型（如图片、PDF、二进制文件）及超大文件（>500KB）点击时自动引导下载到本地查看
- 未保存修改时关闭页面或切换文件会弹出确认提示
- 左侧面板宽度可通过拖拽分割条自由调整

### 编辑 SKILL.md

1. 在文件树中点击 `SKILL.md`（新建技能包后自动选中）
2. 在右侧 CodeMirror Markdown 编辑器中进行编辑
3. 保存后，系统自动解析 YAML frontmatter 中的 `name` 和 `description`，同步更新数据库中的技能包元信息

### 设置分类

在技能包卡片右上角菜单中点击 **分类**，弹出分类选择器：

- 从已有分类中选择
- 支持搜索过滤
- 支持新建分类：在分隔线下方输入新分类名称后点击添加

## 导入技能包

### 导入方式对比

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| **本地装载** | 指定服务器本地目录路径，直接装载目录中的技能包 | 已有技能包目录、开发调试 |
| **ZIP 导入** | 上传 ZIP 压缩包，自动解压并导入 | 离线分享、批量导入 |
| **Git 导入** | 从 Git 仓库拉取技能定义 | 版本管理、团队协作 |

### 覆盖策略

导入时遇到同名技能包需选择策略：

| 策略 | 行为 | 适用场景 |
|------|------|----------|
| **跳过** | 同名技能包保留已有内容，跳过导入 | 防止误覆盖、保留本地修改 |
| **覆盖** | 删除旧内容后导入新内容 | 版本更新、全量同步 |


## 本地装载详解

本地装载从服务器本地目录直接导入技能包。

**工作原理：**

1. 读取指定目录下的技能包目录
2. 解析各目录中 `SKILL.md` 的 YAML frontmatter 获取 `name`、`description`
3. 将技能包目录安装到平台工作空间 `.apboa/skills/` 下
4. 将技能包元数据及附属目录下的文件持久化到数据库

**目录结构要求：**

```
<指定路径>/
  skill-a/
    SKILL.md          # 须含 YAML frontmatter: name, description
    scripts/           # 脚本文件
    examples/          # 示例文件
    references/        # 引用文件
  skill-b/
    SKILL.md
    ...
```

若指定路径下存在 `skills/` 子目录，系统优先从该子目录加载。当本地路径与平台工作空间路径相同时，系统跳过文件复制，仅更新数据库记录。


## ZIP 压缩包导入详解

**工作原理：**

1. 将上传的 ZIP 文件解压到临时目录（`.apboa/temp/`）
2. 读取解压后目录中的技能包定义
3. 根据覆盖策略安装到工作空间
4. 安装完成后自动清理临时目录

**ZIP 包结构要求：** 同本地装载，支持 `skills/` 在根目录或外层目录下。

:::warning
ZIP 导入过程中的临时目录会在完成后自动清理，不占用额外磁盘空间。
:::


## Git 仓库导入详解

**工作原理：**

1. 将远程仓库克隆到临时目录
2. 若仓库根目录存在 `skills/` 子目录则优先加载，否则从根目录加载
3. 根据覆盖策略安装到工作空间
4. 安装完成后自动清理临时目录

**仓库目录结构：** 推荐将技能包统一放置在仓库根目录的 `skills/` 子目录下。

:::info
若仓库中不存在 `skills/` 子目录，系统将仓库根目录视为技能包目录直接加载。
:::


## 在智能体中使用

在智能体配置的「工具与能力」步骤中：

1. 在 **技能包** 区域，按分类浏览可用技能包
2. 勾选需要的技能包（多选）
3. 技能包关联的工具会自动注册到智能体的工具集中

## 最佳实践

### SKILL.md 编写建议

SKILL.md 是模型理解技能能力的关键入口，建议包含：

- **技能目标** — 该技能解决什么问题
- **使用场景** — 什么情况下应该使用该技能
- **操作步骤** — 使用该技能的典型步骤
- **注意事项** — 使用限制和注意点

### 文件管理建议

- 将脚本放入 `scripts/`、示例放入 `examples/`、参考文档放入 `references/` 目录，便于系统自动识别入库
- 二进制资源文件（图片、PDF 等）可上传到任意目录，系统会自动下载引导而非尝试在线预览
- 非白名单文件类型无法在线编辑，请下载到本地修改后重新上传

### 工具关联建议

- 将逻辑上相关的工具组合到同一技能包
- 避免将不相关工具放入同一技能包
- 确保技能包关联的工具都已启用


## 目录结构规范总结

为确保技能包可正常导入和使用，请遵循以下规范：

1. 每个技能包使用独立目录
2. 技能定义文件统一命名为 `SKILL.md`（全大写）；小写 `skill.md` 导入时自动重命名
3. `SKILL.md` 须含 YAML frontmatter，至少包含 `name` 与 `description` 字段
4. 脚本文件放入 `scripts/` 子目录
5. 示例文件放入 `examples/` 子目录
6. 引用文件放入 `references/` 子目录
7. 多个技能包统一放在 `skills/` 父目录下（推荐）


## 导入后验证

导入完成后建议进行以下检查：

1. 在技能包管理页面确认技能包列表已更新
2. 进入技能包编辑器，确认文件结构完整
3. 在智能体中关联该技能包，验证运行时工具注册是否正常
