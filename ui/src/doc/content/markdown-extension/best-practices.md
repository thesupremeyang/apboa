# 三、最佳实践

## 3.1 你需要创建独立引擎吗？

:::info 不需要创建独立引擎的情况（大多数）
- 只是想修改图片、代码块等元素的渲染方式 → 直接改 handler 文件
- 想添加新的容器语法 → 创建 -extension.ts 文件，自动发现
- 项目里用 `renderMarkdown()` 函数 → 默认引擎够用了
:::

:::info 需要创建独立引擎的情况（少数）
- 你想在某个组件里用完全不同的渲染配置，不影响全局
- 你需要多个不同的 Markdown 引擎实例
:::

## 3.2 常见问题

**Q: 我创建了新的容器扩展文件，为什么不生效？**

A: 检查三点：
1. 文件名是否以 `-extension.ts` 结尾
2. 是否导出了 `extensionModule` 对象
3. `meta.enabled` 是否为 `true`

**Q: 我修改了 handler 文件，为什么不生效？**

A: 可能是缓存问题，重启开发服务器试试：`pnpm dev`

**Q: 容器内容里的 Markdown 语法没有被解析？**

A: 你需要在 `setup` 函数里设置 `innerRender`，参考 1.5 节。


**Q: 自定义的 HTML 标签被过滤了？**

A: 引擎默认启用 DOMPurify 做 XSS 防护。需要配置白名单。

## 3.3 注意事项

:::warning 注意事项
- **文件命名**：容器扩展文件必须以 `-extension.ts` 结尾
- **名称唯一**：`meta.name` 不能重复
- **必须 use()**：创建独立引擎后，注册扩展必须调用 `engine.use()`
:::
