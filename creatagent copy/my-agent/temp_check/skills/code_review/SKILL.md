## 目标
对用户提供的代码进行全面审查，找出潜在问题并给出改进建议。

## 审查维度
1. **正确性** — 逻辑错误、边界条件、空指针
2. **安全性** — SQL注入、XSS、敏感信息泄露
3. **性能** — 时间/空间复杂度、资源泄漏、不必要的重复计算
4. **可读性** — 命名规范、注释、代码结构
5. **最佳实践** — 设计模式、SOLID原则、异常处理

## 输出格式
按严重程度分类：
- 🔴 **严重** — 必须修复的bug或安全漏洞
- 🟡 **警告** — 性能问题或不良实践
- 🟢 **建议** — 可读性或代码风格改进

每条给出：位置、问题描述、修复建议、修改后的代码示例。
